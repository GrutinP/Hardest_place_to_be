# 開発ロードマップ

---

## フェーズ概要

| フェーズ | テーマ | ゴール |
|---|---|---|
| Phase 1 | シナリオ完成 | 最初から最後まで通して読める |
| Phase 2 | 演出実装 | 「見せられる」動きが揃う |
| Phase 3 | UI整備 | ゲームとして遊べる状態になる |
| Phase 4 | 本素材差し替え | リリース品質になる |
| Phase 5 | 仕上げ・リリース | 配布できる状態になる |

---

## Phase 1 ― シナリオ完成

**目標：全ルートを仮素材で通しプレイできること**

優先度は上から順。

1. **残りシナリオの流し込み**（全て仮素材・SEコメントアウトで可）
   - 05_bad01_a / 06_bad01_b
   - 10_bad03 / 11_route_a / 12_route_b
   - 13〜16（コンビニ・告白ルート）
   - 17_quiz / 18_quiz_main
   - 20_gate_common / 21_ed_a / 22_ed_b / 23_ed_c / 24_ed_1 / 25_true_end

2. **分岐・変数の動作確認**
   - f.chara_saved の加算ロジック（③A / ⑥A / ⑧A）
   - ED分岐（20_gate_common の jump 条件）
   - 魚座分岐（名前・立ち絵・雨の方向・背景スクロール方向）

3. **tips機能の基礎実装**
   - `;tips：` コメントをプラグインまたはスクリプトで拾う仕組み
   - 魚座プレイヤー限定で心の声を読める機能

4. **不足仮素材の用意**
   - トラックアイコン / 犬アイコン
   - chara_pu_effect.png
   - bg_conbini / bg_eki_soto

---

## Phase 2 ― 演出実装

**目標：アイコン・エフェクト演出が動くこと**

シナリオが通しで読めるようになってから着手する。
演出ごとに仮グラで動作を確認し、後から本素材に差し替える。

- トラックアイコン演出（04_bad01 登場 / 05_bad01_a 消去）
- 犬アイコン演出（07_bad02 接近 / 08_bad02_a・09_bad02_b 衝突）
- 水しぶき演出（05_bad01_a）
- 風・枝葉・ゴミ演出（10_bad03）
- キャラ出現エフェクト（03_prologue）

---

## Phase 3 ― UI整備

**目標：ゲームとして一通り操作できること**

演出が揃ってから着手。UIは仮グラ・仮フォントで実装し、Phase 4で本素材に差し替える。
実装順の目安：

1. **テキストウィンドウ** （ゲームプレイの基盤）
   - メッセージウィンドウ・フォントの変更
   - ボタン：Qsave / Qload / backlog / メニュー

2. **タイトル画面** （ゲームの入口）
   - レイアウト・ボタン（はじめから / つづきから / コンフィグ / おまけ）

3. **メニュー画面**
   - ボタン：セーブ / ロード / コンフィグ / タイトルに戻る
   - 背景用傘アニメーション（仮実装）

4. **コンフィグ画面**
   - ボイス音量スライダー追加 / 不要項目削除
   - 傘アイコン（開/閉）

5. **セーブ・ロード画面**
   - 配置・見た目の変更（機能は変えない）

6. **選択肢グラフィック**
   - 赤傘・緑傘 / ホバーアニメ / クリック反応

7. **おまけ画面**
   - tipsリスト / スタッフあとがき

---

## Phase 4 ― 本素材差し替え

**目標：全素材がリリース品質になること**

UIの枠組みが完成してから着手。素材担当者との連携が必要。

- **キャラ本素材**（担当者へ仕様渡し → 受け取り次第差し替え）
- **背景写真**撮影・加工
- **BGM / SE**（作曲・録音 → 差し替え）
- **ボイス**（フルボイスか一部かを決定 → 録音 → 差し替え）
- **タイトル画面の絵**
- **スチルCG**（25_true_end）
- **UIグラフィック一式**
  - 選択肢傘（赤・緑）
  - メニューボタン / テキストWボタン
  - 傘アイコン（コンフィグ用 開/閉）
  - タイトルロゴ
- **傘アニメーション**（メニュー背景・コンフィグアイコン 本素材版）

---

## Phase 5 ― 仕上げ・リリース

**目標：配布できる状態にすること**

1. **ローカライズ**（対象言語・範囲を決定してから）
   - テキスト抽出 → 翻訳 → TyranoScript に差し込み

2. **通しプレイ・デバッグ**
   - 全ルート（EDa / EDb / EDc / true_end）を通してプレイ
   - フラグ・分岐の動作確認
   - 誤字脱字チェック

3. **リリースビルド**
   - TyranoScript の書き出し設定確認
   - 配布形式（Web / exe / 同梱圧縮等）の決定

---

## 依存関係メモ

- Phase 1 が終わるまで Phase 2 は着手しない（シナリオが未確定だと演出の差し込み位置が変わる）
- Phase 3 のUI実装は Phase 2 と並行して進められるが、テキスト本文が確定していないとおまけ画面のtipsリストが作れない
- Phase 4 の素材差し替えは Phase 3 のUI枠組みがないと入れ場所がない
- ローカライズはテキストが完全に確定してから（Phase 4 完了後）
