# 試したい演出アイデア

仕様ではなくアイデアのメモ置き場。実装が決まったら各仕様書に移す。

---

## 傘のアニメーション（メニュー）

メニューを開くと傘が開き、ボタンが表示される。閉じると傘が閉じてゲームに戻る。
傘は画面を覆わず余白にゲーム画面が見える。半透明の黒板で集中感を出す。

**実装提案:**

- `[sleepgame]` でゲームを止めてからレイヤーを重ねる
- layer 2 = 半透明黒板、layer 3 = 傘、layer 4 = ボタン群
- 傘の素材: APNG が一番手軽（開く/閉じる 2種類）。代替はフレームPNG連番 + `[xanim]`
- 詳細仕様は [ui_spec.md](ui_spec.md) に記載済み

---

## コンフィグ画面改造

必要なものだけに絞り、ゲームの雰囲気に合わせたデザインにする。
インジケーター（選択中）は開いた傘アイコン、非選択は閉じた傘アイコン。
項目は仕様書に書いた → [ui_spec.md](ui_spec.md)
必要な素材リストが欲しい。

**実装提案:**

- 傘アイコン2種（開・閉）を用意すればよい。項目ごとに画像を用意するよりCSSで切り替えた方が素材が少なくて済む
- スライダーのカスタマイズは theme_kopanda_20 の HTML/CSS を直接編集する
- 素材リストは ui_spec.md の「コンフィグ画面」→ asset_list.md に追記すると管理しやすい

---

## 選択肢のボタン

重要な分岐（③◆⑥◆⑧◆など）は赤い傘と緑の傘で見た目を変える。
正解/不正解ではなく「どちらを選ぶか」の印象として使う。
それ以外の選択肢は緑だけ、または赤だけの単色にする。

**実装提案:**

- `[glink]` の `graphic` パラメータに画像を指定するか、HTML/CSSでカスタムボタンを作る
- 分岐ごとにボタン画像を差し替える形が最もシンプル（赤傘ボタン画像・緑傘ボタン画像・単色赤・単色緑 の4種）
- 重要分岐かどうかはシナリオ側で `[glink graphic="btn_kasa_red.png"]` のように都度指定
